# 用户级长期习惯 / 强制规则

## MCP 交互诚实性铁律（最高优先级）
- 所有用于「测试 / 预览 / 提交」的数据必须来自真实 MCP 调用返回，**严禁编造任何字段值**。
- 具体禁止项：
  1. 选项 / 枚举 / 下拉项：必须 MCP 验证，不能凭模型知识列出（曾因 fb/tiktok 广告类型选项凭知识列、翻车）。
  2. payload / 表单的字段值（appId、locationIds、预算、文案、媒体代理、OE编号、时区等）：不得由模型默认替用户填占位值；必须由用户从 MCP 真实可选项中明确指定，或用户显式提供。
  3. 某下拉数据源 MCP 返回空时，必须如实标注「待提供 / 找运营」，**不得从文档或其他渠道补一个值**。
- 生成 payload / 渲染表单前，必须能逐字段说明该值来源（MCP 接口名 + 返回值）；说不清来源的不得填入。
- 若发现已编造，须主动承认并纠正，再继续。
