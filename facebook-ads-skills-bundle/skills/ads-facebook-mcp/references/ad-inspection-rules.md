# AI 广告巡检规则（ad-inspection-rules）— Facebook / Meta

本文件为「模块二：AI 广告巡检」的详细规则。MCP 调用遵循 `references/mcp-client.md`（两步滤波法见 §4）。

> 与 TikTok 巡检规则同构，六类异常优先级与阈值一致；排查 / 止损 / 优化建议全面适配 Meta 平台特性（Advantage+ 系列、学习期、Pixel + Conversions API、Advanced Matching、Meta 版位等）。

---

## 1. 功能与职责边界

- **功能**：提醒广告数据变化趋势，辅助及时调整。
- **职责**：巡检目标、异常指标、阈值规则、建议动作、多异常处理优先级。
- **覆盖**：花费异常、转化异常、CPA 异常、ROAS 异常、预算耗尽、素材疲劳、Meta 政策合规与账户质量。

---

## 2. 六类异常（按优先级）

| 优先级 | 异常类型 | 核心判断 |
|--------|----------|----------|
| 1 | 花费异常 | 花费过快 / 过慢 / 突然不花 / 暴涨 |
| 2 | 转化异常 | 转化下降 / 归零 / 有点击无转化 |
| 3 | CPA 异常 | CPA > 目标×1.3、CPA > 历史×1.5、有花费无转化 |
| 4 | ROAS 异常 | ROAS < 目标×0.7、ROAS < 历史×0.6、有消耗无收入 |
| 5 | 预算耗尽 | 日预算≥70%、日预算≥95%、总预算≥90%、归零停投 |
| 6 | 素材疲劳 | CTR < 历史×0.7、CVR < 历史×0.7、连续≥7天且 CTR 降≥30%、频次过高 |
| 7 | Meta 政策合规与账户质量 | 账户质量限流、政策违规、投放受限（见 §7 专项检查） |

> 六类投放异常（1–6）按优先级处理；第 7 类为账户级专项检查，可独立于投放异常触发，亦可与投放异常并存。

---

## 2.5 学习期全局前置过滤（贯穿六类异常判断）

Meta 广告的学习期（Learning Phase）是**贯穿所有异常判断的全局前置条件**。Ad Set 在创建后或重大修改后，需累计约 50 次优化事件（约 7 天）才能退出学习期。

**学习期识别**：Ad Set 状态为 `LEARNING`，且 7 天内累计优化事件 < 50。

**豁免规则（避免误报）**：
- 处于学习期的 Ad Set，单日成本波动在 ±25%~50% 内属正常，**不触发** CPA 异常（§3）、ROAS 异常（§4）报警，除非已触及止损线（CPA > 目标×2 或 ROAS < 目标×0.5 且无转化）。
- 花费异常（§1）在学习期内的"暴涨"需结合转化判断：仅当花费 ≥ 目标 CPA×1.5 且无转化时才视为异常，否则按"学习期探索放量"处理。
- 素材疲劳（§6）在学习期前 3 天内不判定为疲劳（需足够样本）。

**学习期重置检测**：若最近 7 天内对 Ad Set 做过以下任一操作，系统会重置学习期，成本波动属正常，不应误报：
- 预算调整幅度 > ±50%
- 修改定向（受众、地区、年龄、版位）
- 修改出价策略或出价值
- 新增 / 删除 / 替换素材
- 修改优化目标或转化事件

→ 巡检应在异常卡片中标注：「⚠️ 该 Ad Set 于 X 前因（操作）重置学习期，当前成本波动为学习期正常现象，建议观察 3–7 天再干预，避免再次重置」。

**学习失败识别**：Ad Set 长期处于 `LEARNING` 状态（> 7 天）且累计优化事件仍 < 50，识别为"学习失败"专项提示（非通用异常），建议：
- 检查预算是否低于学习所需（深层转化建议 ≥ 10×CPA、浅层 ≥ 50×CPA）
- 检查定向是否过窄、素材是否过多导致系统分散学习
- 考虑合并 Ad Set、简化结构或提高预算

## 2.6 行业基准阈值细化（避免一刀切）

Meta 官方未公开统一异常阈值，业内经验基准差异较大。巡检采用"双层基线"机制：

1. **第一层：账户历史基线（优先）** — 默认以该 Ad Set / 账户过去 7–14 日的历史均值作为阈值基准（即本文件 §7 中各异常规则的"历史均值×系数"逻辑）。
2. **第二层：行业基准（补充）** — 当历史样本不足（新 Ad Set / 学习期 / 冷启动）时，参考行业经验基准：

| 行业 / 目标 | CTR 参考 | ROAS 参考 | CPA 波动容差 |
|-------------|----------|-----------|--------------|
| 电商 Feed 投放 | 1%–2% | 2–4 | ±30% |
| 游戏 App 推广 | 0.5%–1% | 1.5–2.5 | ±40% |
| Lead 表单类 | 0.8%–1.5% | — | ±35% |
| 本地到店 / 品牌 | 0.5%–1% | — | ±40% |

**阈值调整机制**：巡检输出异常时，若采用行业基准而非账户历史，应在卡片中注明「行业基准：CTR 低于电商平均 1%–2%」。避免把"低于行业均值但自身稳定"误报为异常。

## 3. 巡检流程

发现异常 → 检测阈值规则 → 根因分析（排查 + 止损建议 + 优化建议）。

多异常同时出现时，按优先级顺序展示「排查 / 止损建议 / 优化建议」。

## 4. MCP 执行（两步滤波法，< 10s）

统一使用 `references/mcp-client.md` §4 的 `list_active_advertisers_with_today_data`：

1. `list_advertisers` → 过滤 `campaignsCount > 0`
2. 并行 `get_daily_metrics(level=ACCOUNT)` → 找今日真有数据的账户（通常仅少数）
3. **只对有数据的账户**拉 7 天历史做阈值对比

> ⚠️ 不调 `sync_advertiser_data`（交互场景，见 mcp-client §8）
> ⚠️ `as_completed` 必须 `timeout=30`
> ⚠️ 单次 `timeout=15`

## 7. 六类异常详细规则

### 1. 花费异常

**关键指标**：今日花费 / 小时花费 / 预算消耗进度 / 历史同期花费 / 花费环比变化率

**正常条件**：今日花费接近预算进度 / 小时花费没有明显偏离历史均值 / 广告状态正常 / 账户余额充足 / 预算未耗尽

**异常规则**：
- A. 花费过快：当前花费 > 预算 × 当日时间进度 × 1.3，且当前花费 > 过去 7 日同期平均花费 × 1.5
- B. 花费过慢：当前花费 < 预算 × 当日时间进度 × 0.5，且广告已投放超过 3 小时，且广告状态正常
- C. 突然不花费：过去 3.5 小时花费 = 0，且过去 24 小时有花费，且 Ad Set 状态为 ACTIVE
- D. 花费暴涨：今日花费 ≥ 日预算的 25%

#### 1.1 花费过快 / 消耗暴涨
**优先排查**
- 检查是否刚刚提高预算、放宽定向、切换出价策略、复制 Ad Set 或新增素材。
- 检查 CPM 是否上涨，判断是竞价环境变贵，还是系统突然放量。
- 检查转化是否同步增长；如果只涨花费、不涨转化，优先按低效放量处理。
- 检查是否使用 Lowest Cost（`LOWEST_COST_WITHOUT_CAP`），Meta 该策略目标是尽量在预算内获取更多结果，不保证 CPA 稳定。Meta 官方也提示，学习期或高竞争期 Cost per Result 可能出现日波动。
- 检查 Ad Set 是否处于学习期（`LEARNING` 状态），学习期波动属正常。

**止损建议**
- 若 CPA / ROAS 同时恶化：立即降低预算 20% - 30%，不要一次性大幅砍到很低。
- 若单个 Ad Set 花费暴涨但转化差：暂停低效素材或低效 Ad Set。
- 若是新 Ad Set 学习期内暴涨：先看是否已花费超过目标 CPA 的 1.5 - 2 倍且无转化；达到后再处理。Meta 学习期需 7 天 / 50 次转化，过早干预会重置学习。
- 若使用 CBO（Advantage+ Campaign Budget）：检查预算是否被自动集中到表现差但出价高的 Ad Set。

**优化建议**
- 将预算从异常 Ad Set 转移到 CPA 稳定、ROAS 稳定的 Ad Set。
- 如果当前用 Lowest Cost 且 CPA 失控，可测试 Cost Cap（`COST_CAP`），设置可接受的目标 CPA。
- 对高消耗素材做拆解：看 CTR、CVR、Hook 留存、落地页转化，判断是素材吸引错人，还是页面承接差。
- 若预算上调导致波动，后续扩量建议每次提高 20% - 50%，避免频繁大幅调整。Meta 官方建议学习期后预算调整保持在此前预算 50% 范围内，并至少等待 2 天再改定向、出价或预算。

#### 1.2 花费过慢 / 不起量
**优先排查**
- 检查 Ad Set、Ad、素材是否均已 ACTIVE 并审核通过。
- 检查预算是否过低，尤其是转化广告是否低于系统学习所需预算。
- 检查出价是否过低，Cost Cap 下低出价很容易导致低消耗。
- 检查定向是否过窄，例如年龄、兴趣、地区、设备、自定义受众叠加太多。
- 检查素材 CTR 是否偏低，CTR 低会影响系统分发意愿。
- 检查是否未通过 Meta 广告政策审核（账户被限 / 素材被拒）。

**止损建议**
- 若连续 3 小时以上无花费，且审核、预算、账户余额均正常：优先提高 Cost Cap 出价 20%。
- 若定向过窄：放宽兴趣、自定义受众、年龄或版位；建议开启 Advantage+ Audience 自动扩展。
- 若预算过低：提高日预算到目标 CPA 的 5 - 10 倍以上；Meta 对 Cost Cap 的建议是预算至少为 CPA bid 的 10 倍，以支持投放与学习。

**优化建议**
- 新 Ad Set 建议用较宽定向 + Advantage+ Audience，让系统先探索有效人群。
- 用 Lowest Cost 起量，再在 CPA 稳定后测试 Cost Cap 控成本。
- 增加 3 - 5 条差异明显的素材，提高系统探索空间。Meta 官方建议每个 Ad Set 配置多条不同素材，每个 Campaign 配置 3 - 5 个差异化 Ad Set。
- 如果多个 Ad Set 都不起量，优先检查账户层级问题：余额、付款失败、素材审核、行业政策、Pixel / Conversions API 回传。

### 2. 转化异常

**关键指标**：转化数 / 转化率 / 点击数 / 消耗

**正常条件**：转化数接近 3 日历史均值 / CVR 没有明显下降 / 点击量和消耗正常

**异常规则**：
- A. 转化数下降：今日转化数 < 过去 7 日平均转化数 × 0.5，且今日花费 ≥ 过去 7 日平均花费 × 0.7
- B. 转化归零：今日转化数 = 0，且今日花费 ≥ 目标 CPA × 1.5
- C. 点击正常但无转化：今日点击数 ≥ 过去 7 日平均点击数 × 0.8，且今日转化数 = 0

#### 2.1 花费正常但转化下降
**优先排查**
- 检查 **Meta Pixel / Conversions API (CAPI)** 是否正常回传。建议在 Meta 事件管理器查看事件接收状态。
- 检查转化事件是否被修改，例如从 Purchase 改成 Add to Cart，或事件去重异常。
- 检查落地页是否打开慢、报错、断链、支付失败、商品缺货。
- 检查 CTR 是否稳定；如果 CTR 正常但 CVR 下滑，多半是页面、价格、商品、支付、库存或回传问题。
- 检查 CVR 是否从某个小时开始突然断崖式下跌，若是，优先判断技术问题。

**止损建议**
- 若转化归零且花费已达到目标 CPA 的 1.5 - 2 倍：暂停该 Ad Set 或降低预算。
- 若账户多 Ad Set 同时转化下降：不要先调广告，先查埋点、页面、支付、库存。
- 若只有单个素材或单个 Ad Set 异常：暂停异常对象，预算转移到稳定对象。

**优化建议**
- 对 Web 转化广告，建议同时使用 **Meta Pixel + Conversions API**，Meta 官方说明该组合可以从浏览器和服务器两端发送事件，有助于提升事件量、测量和广告表现，并增强 iOS 14.5+ 信号衰减下的归因。
- 开启 **Advanced Matching**，提升用户匹配和归因质量；Meta 官方建议在可行情况下同时使用 Manual 和 Automatic Advanced Matching。
- 如果是 App 投放，检查 MMP 事件、SKAdNetwork (SAN) 集成、postback、归因窗口、事件映射是否正常。Meta 官方也强调 Purchase 等事件需要传递 value / revenue 才能支持 VBO 和 ROAS 计算。
- 如果转化下降来自 CVR 下滑，优先优化落地页：首屏卖点、加载速度、价格展示、信任背书、支付流程、优惠券、库存状态。

#### 2.2 点击正常但无转化
**优先排查**
- 检查素材是否「吸引点击但不筛选人群」，例如夸张 Hook、误导性价格、泛娱乐内容过重。
- 检查落地页与广告承诺是否一致。
- 检查商品价格、运费、优惠、支付方式是否与广告信息一致。
- 检查跳转链路：Meta → 落地页 → 商品页 → 加购 → 支付。

**止损建议**
- 花费达到目标 CPA 的 2 倍仍无转化：暂停素材或 Ad Set。
- CTR 很高但 CVR 极低：优先暂停该素材，避免继续吸引低质量点击。
- 若多个素材点击正常但均无转化：暂停扩量，先修复页面或事件回传。

**优化建议**
- 重写素材前 3 秒，让用户在点击前理解价格、场景、适用人群和核心利益点。
- 增加「人群筛选型」表达，例如「适合 XX 人群」「不适合 XX 情况」「预算在 XX 以上再看」。
- 落地页首屏与素材首屏保持一致，避免用户点击后心理落差。
- 对高点击低转化素材，保留 Hook 形式，但替换卖点、价格说明、CTA 和目标人群表达。

#### 2.3 iOS 14.5+ 归因衰减专项检查

自 iOS 14.5+ ATT 隐私政策生效以来，Meta 归因信号显著衰减，转化异常排查**必须纳入归因维度**：

**优先排查**
- 检查转化下降是否伴随 **Pixel 单通道事件量下降**：若仅依赖 Meta Pixel（浏览器端），iOS 用户事件可能缺失 15%–30%，应确认是否同时启用 **Conversions API（CAPI，服务端）** 双通道回传。
- 检查 **归因窗口**：Meta 默认归因窗口缩短（如 7 天点击 / 1 天浏览），长周期转化（高客单、教育、B2B）的归因延迟会更大，转化下降可能是归因窗口未覆盖而非真实下滑。
- 检查 **Aggregated Event Measurement（AEM）** 配置：iOS 14+ 下每个 domain 仅能配置优先事件，若购买事件未设为高优先级，会丢失低优先级事件。
- 检查 SKAdNetwork（App 投放）：iOS 14+ App 推广需确认 SKAdNetwork postback、MMP（如 AppsFlyer / Adjust）集成、归因窗口与事件映射正常。

**止损建议**
- 若确认是归因衰减（CAPI 缺失）导致"转化归零"：先补 CAPI 双通道，**不要立即暂停广告**——真实转化可能仍在发生但未被回传。
- 启用 CAPI + Advanced Matching 后，观察 3–7 天再判断是否真实下滑。

**优化建议**
- Web 投放强制 **Pixel + CAPI 双通道**，并在事件管理器核对两端事件量一致性。
- App 投放启用 **SKAdNetwork + MMP + CAPI** 三重归因，优先用 `APP_INSTALLS_AND_OFFSITE_CONVERSIONS` 优化目标兼顾安装与后端转化。
- 对高客单 / 长周期转化，放宽异常判定窗口至 14 天，避免归因延迟误报。

### 3. CPA 异常（转化成本异常）

**关键指标**：目标 CPA / 历史 CPA / 转化数 / 花费

**正常条件**：CPA 接近过去 7 日均值

**异常规则**：
- A. CPA 高于目标：当前 CPA > 目标 CPA × 1.3，且转化数 ≥ 3
- B. CPA 相比历史低下：当前 CPA > 过去 7 日 CPA 均值 × 1.5，且转化数 ≥ 3
- C. 无转化 CPA 风险：当前花费 ≥ 目标 CPA × 1.5，且转化数 = 0

#### 3.1 CPA 高于目标 CPA
**优先排查**
- 拆解 CPA：`CPA = CPC / CVR`，所以要判断是 CPC 变贵，还是 CVR 变差。
- 如果 CPC 上升：检查 CPM、CTR、竞价环境、素材吸引力。
- 如果 CVR 下降：检查落地页、事件回传、商品、价格、支付链路。
- 检查是否刚调预算、出价、定向或素材，导致系统重新进入学习期。

**止损建议**
- CPA > 目标 CPA × 1.5，且转化数 ≥ 3：降低预算 20% - 30% 或暂停低效 Ad Set。
- CPA > 目标 CPA × 2：暂停，除非该 Ad Set 仍处于早期学习期且样本过少。
- 无转化但花费达到目标 CPA × 2：暂停或换事件、换素材、换定向。
- 对 Cost Cap Ad Set，不建议 CPA 刚变差就立刻大幅降 bid；Meta 官方提示，如果 CPA / ROAS 未达目标，立即降低出价可能导致消耗大幅下降。

**优化建议**
- 若 CTR 下降导致 CPA 升高：更换素材 Hook、前 3 秒卖点、封面、字幕、达人出镜方式。
- 若 CVR 下降导致 CPA 升高：优化落地页、商品页、价格、优惠、支付、事件回传。
- 若 CPM 上升导致 CPA 升高：拓宽定向、测试更多国家 / 人群 / 版位，避免小人群竞争。
- 若学习期 CPA 波动：减少频繁改动，至少观察 24 - 48 小时，除非已达到止损线。
- 若稳定 Ad Set CPA 变差：先降预算，不要直接删除；同时复制历史优质结构做新 Ad Set 测试。
- 若 Cost Cap 不起量但 CPA 合理：可小幅提高 bid 10% - 20% 换取更多量级。
- 若 Lowest Cost 起量快但 CPA 不稳：测试 Cost Cap 或混合投放结构，一组冲量，一组控成本。

#### 3.2 CPA 突然升高但 ROAS 没有下降
**优先排查**
- 检查客单价 AOV 是否上升。
- 检查购买事件数减少但单笔金额变高。
- 检查优化目标是 Purchase（数量）还是 Value / ROAS（价值）。
- 检查归因延迟，尤其是高客单价产品。

**优化建议**
- 如果 ROAS 达标，不建议只因 CPA 升高立即暂停。
- 将判断优先级从 CPA 切换为 ROAS / 毛利 ROI。
- 对高客单素材单独建组，允许更高 CPA，但设置 ROAS 或利润底线。
- 拉新广告看 CPA，电商购买广告优先看 ROAS、AOV、毛利和退款率。

### 4. ROAS 异常

**关键指标**：目标 ROAS / 历史 ROAS / GMV / 花费 / 购买数

**正常条件**：ROAS 接近历史均值

**异常规则**：
- A. ROAS 低于目标：当前 ROAS < 目标 ROAS × 0.7，且花费 ≥ 日预算的 30%
- B. ROAS 相比历史下降：当前 ROAS < 过去 7 日 ROAS 均值 × 0.6，且花费 ≥ 过去 7 日平均花费 × 0.7
- C. 有消耗无收入：当前花费 ≥ 目标客单价 × 0.5，且 conversion_value = 0

#### 4.1 ROAS 低于目标
**优先排查**
- 拆解 ROAS：`ROAS = 转化价值 / 花费 = CVR × AOV / CPC`，判断是 CPC 变贵、CVR 变差，还是 AOV 下降。
- 检查商品折扣、客单价、运费、库存、优惠券是否变化。
- 检查 Purchase value 是否正常回传，currency 是否一致。
- 检查广告是否吸引了低购买力人群。

**止损建议**
- ROAS < 目标 ROAS × 0.5，且花费已达到有效样本：降低预算或暂停。
- ROAS 连续 2 天低于目标 70%：减少预算 20% - 50%，预算转移到高 ROAS 组。
- 有花费无 GMV：优先检查 Purchase 事件、value 参数、支付链路和库存；确认不是回传问题后再暂停广告。

**优化建议**
- 若 AOV 下降：测试高客单套餐、组合装、满减门槛、加价购。
- 若 CVR 下降：优化商品页、首屏信任背书、评论、物流、退换政策、支付体验。
- 若 CPC 上升：更新素材、拓宽定向、测试新 Hook 和新场景。
- 若购买人数有但 ROAS 低：优先推高毛利、高客单、高复购商品。
- 若使用 **Advantage+ Shopping Campaign** 或商品目录广告（DPA）：检查商品卡、标题、价格、主图、库存、评价、物流时效。
- 若 App 或 Web 价值优化：确保 value / revenue 正常传回。Meta 官方明确提到，购买或相关应用事件需要发送 value 和 revenue，才能支持 VBO（Value-Based Optimization）和 ROAS 计算。建议优化目标用 `VALUE`。
- 将低 ROAS 素材按「吸引错人」和「承接不佳」分类：
  - CTR 高、ROAS 低：素材吸引力强但人群不精准。
  - CTR 低、ROAS 低：素材本身竞争力不足。
  - CTR 正常、CVR 低：页面或商品承接问题。

#### 4.2 ROAS 突然下降但 CPA 正常
**优先排查**
- 检查客单价是否下降。
- 检查是否大量低价商品成交。
- 检查是否优惠力度过大，导致 GMV 或利润被稀释。
- 检查 Purchase value 是否漏传或传错。
- 检查是否只优化购买数量（`OFFSITE_CONVERSIONS`），没有优化购买价值（`VALUE`）。

**优化建议**
- 不要只看购买数，增加 AOV、毛利、退款率维度。
- 优先放量高 AOV 素材、高毛利 SKU、高意向人群。
- 将低客单商品作为引流组，高客单商品作为利润组，分开预算和目标。
- 对低 ROAS 但 CPA 正常的 Ad Set，不一定立即暂停；先看是否能通过商品组合、加购优惠和页面排序提升客单价。

### 5. 预算异常（预算耗尽）

**关键指标**：花费 / 日预算 / Campaign 预算 / Ad Set 预算 / 投放状态

**正常条件**：预算剩余充足 / 广告仍在正常消耗

**异常规则**：
- A. 日预算即将耗尽：今日花费 / 日预算 ≥ 70%
- B. 日预算已耗尽：今日花费 / 日预算 ≥ 95%
- C. 总预算即将耗尽：累计花费 / 总预算 ≥ 90%
- D. 预算耗尽导致停止投放：预算剩余 = 0

#### 5.1 日预算过早耗尽
**优先排查**
- 检查预算耗尽时间：上午耗尽、中午耗尽、晚间耗尽的处理方式不同。
- 检查预算耗尽前的 CPA / ROAS 是否达标。
- 检查是否因为预算太低导致系统无法稳定学习（学习期需 50 转化 / 7 天）。
- 检查 Campaign Budget（CBO）和 Ad Set Budget 的分配是否合理。

**止损建议**
- 如果 CPA / ROAS 不达标：不要加预算，先控成本、修素材、修页面。
- 如果 CPA / ROAS 达标且预算过早耗尽：可以逐步增加预算。
- 如果上午就耗尽且效果好：提高预算 20% - 50%，或复制扩量测试。
- 如果下午或晚上耗尽且效果稳定：可以适度加预算，但要观察次日成本波动。

**优化建议**
- 对表现好的 Ad Set，优先小步加预算，而不是一次性翻倍。
- 将预算从低效 Ad Set 转移到高转化、高 ROAS、高稳定性的 Ad Set。
- 新 Campaign 初期可使用 **CBO（Advantage+ Campaign Budget）** 让系统自动分配预算到高效 Ad Set；Meta 官方建议 CBO 在 Campaign 层级设预算，系统优化各 Ad Set 分配。
- 转化广告预算过低时，适当提高预算以支持学习。Meta 官方给出的预算参考：深层转化 Campaign 建议 10 × current CPA，安装 Campaign 建议 20 × current CPA，浅层转化或高漏斗事件建议 50 × current CPA / CPR。
- 对稳定 Ad Set，预算调整后至少观察 1 - 2 天，不要当天多次调整。

#### 5.2 总预算即将耗尽
**优先排查**
- 检查 Campaign 是日预算（daily_budget）还是总预算（lifetime_budget）。
- 检查是否活动周期还没结束但预算已接近耗尽。
- 检查高效 Ad Set 是否被低效 Ad Set 抢预算（CBO 场景）。

**优化建议**
- 若整体 ROI 达标：追加预算，优先给高效 Ad Set。
- 若整体 ROI 不达标：不要追加总预算，先关停低效组。
- 对活动型 Campaign，建议按活动节奏拆预算：预热期、爆发期、返场期。
- 对再营销 Campaign，预算耗尽不一定要直接加钱，先检查频次和人群规模，避免过度重复触达。
- 对 Meta Shop 或电商广告，追加预算前同步确认库存、物流、价格和优惠券，避免广告能放量但后端承接不足。

### 6. 素材疲劳

**关键指标**：展示次数 / 点击率 / 转化率 / 单次转化成本 / 广告支出回报率 / 频次

**正常条件**：CTR 稳定 / CVR 稳定 / CPA 未明显升高 / 素材仍有新增转化

**异常规则**：
- A. CTR 明显下降：当前 CTR < 过去 7 日 CTR 均值 × 0.7
- B. CVR 明显下降：当前 CVR < 过去 7 日 CVR 均值 × 0.7
- C. 素材生命周期过长：素材连续投放 ≥ 7 天，且 CTR 下降 ≥ 30%
- D. 频次过高：Frequency > 3（Feed）/ > 2（Reels / Stories）且 CTR 同时下降，说明受众重复触达、新鲜度下降

#### 6.1 CTR 下降 / 互动下降
**优先排查**
- 检查素材投放天数、曝光量、频次、CTR 走势。
- 检查前 3 秒播放率、ThruPlay（完整播放率）、点击率。
- 检查是否同一素材在多个 Ad Set 重复投放，导致受众快速疲劳。
- 检查评论区是否出现负面反馈，例如「太贵」「像广告」「骗人的」「看过很多次」。

**止损建议**
- CTR 连续下滑且 CPA 同时升高：降低预算或暂停疲劳素材。
- 频次升高且新用户触达下降：补充新素材，不要继续硬推旧素材。
- 如果只有个别素材疲劳：不要关整个 Ad Set，先替换或新增素材。

**优化建议**
- 按 Meta 各版位原生内容逻辑更新素材：
  - Facebook / Instagram Feed：1:1 或 **4:5**（推荐），≥ 1080×1080，前 3 秒强 Hook。
  - Stories / Reels：**9:16** 竖屏，≥ 1080×1920，原生感、真人、UGC 风格。
  - 配乐、字幕、安全区内展示核心信息。
- Meta 官方建议素材为 mobile-first，包含声音 / 音乐、竖屏、至少 720P、真人元素和不过度精修的风格。
- 优先重做前 3 秒 Hook。Meta 官方建议在前 3 秒强化 Hook 并介绍内容主张。
- 开启 **Advantage+ Creative**：系统自动裁剪 / 增强素材适配各版位，延长素材生命周期。
- 同一卖点做多种表达：痛点型 / 对比型 / 测评型 / 开箱型 / 达人口播。
- 保留高转化素材的结构，不要完全推翻；换开头、换人物、换场景、换 CTA、换前 3 秒文案。

#### 6.2 CVR 下降 / CPA 上升型疲劳
**优先排查**
- 检查素材是否仍能带来点击，但购买意愿下降。
- 检查是否「老素材继续吸引点击，但吸引的是低质量人群」。
- 检查素材承诺是否过强，导致用户进入页面后流失。
- 检查落地页是否与素材表达一致。

**止损建议**
- CTR 正常但 CVR 下滑超过 30%：暂停该素材或降低预算。
- CPA 高于历史均值 1.5 倍且转化数 ≥ 3：换素材，不建议继续加预算。
- 如果 ROAS 同时下降：优先停低 ROAS 素材，保留高 ROAS 素材。

**优化建议**
- 在素材中提前展示价格、使用门槛、适用场景，减少无效点击。
- 将泛流量素材改成强筛选素材，例如「适合 XX 人群」「XX 场景必备」「预算低于 XX 不建议买」。
- 增加信任元素：真实用户评价、达人试用、对比测试、物流保障、退换承诺。
- 按漏斗补素材：
  - 拉新素材：强 Hook、强场景、强痛点。
  - 种草素材：真实体验、测评、细节展示。
  - 转化素材：优惠、限时、评价、对比、CTA。
- 刷新素材时，优先在原 Ad Set 内新增新素材，而不是每次新建 Ad Set。Meta 官方也建议，当刷新素材时，将新素材加入现有 Ad Set，以延长 Ad Set 生命周期并避免重置学习期。
- 把 **Frequency 作为素材疲劳的核心预警指标**：Feed 频次 > 3、Reels / Stories 频次 > 2 即应准备新素材，避免受众过度重复触达导致 CTR 断崖。

### 7. Meta 政策合规与账户质量专项检查

除六类投放异常外，Meta 账户级合规与质量信号常被忽略，却是表现异常的隐藏根因。

**检查项**：
- **账户质量排名（Account Quality Ranking）**：若广告主被标记为低质量（quality ranking 偏低），Meta 会限流、提高竞价成本。巡检应在概览标注「账户质量评分偏低，可能被限流」。
- **政策违规（Policy Violation）**：单条广告 policy violation 会导致整组 / 整账户表现异常；检查广告管理器中"账户内容质量"面板的违规记录。
- **投放受限（Delivery Limited）**：检查 Ad Set 是否处于「学习受限」「受众过小」「出价偏低」等系统限制状态（Delivery 列提示）。
- **审核状态**：素材 / 落地页是否处于审核中、被拒或已撤。

**优先排查**
- 若多个 Ad Set 同时"不起量"或"花费过慢"且无预算 / 出价问题：优先检查账户质量评分与政策合规状态。
- 若 CPM 突然上涨但 CTR 正常：可能是账户质量下降导致系统惩罚性抬价。

**止损建议**
- 发现 policy violation：立即修改违规素材 / 落地页，移除违规表达（夸大、虚假承诺、对比竞品、敏感词）。
- 账户质量偏低：清理低质广告、提升落地页体验、减少用户反馈投诉（隐藏 / 举报），逐步恢复评分。

**优化建议**
- 定期查看 Meta **账户内容质量（Account Center → 账户质量）** 面板，提前处理违规预警。
- 素材避免"硬广感"：减少过度促销、制造紧迫感的话术，提升原生度。
- 落地页加载速度、移动端适配、信任元素（隐私、退货、客服）直接影响质量评分。
